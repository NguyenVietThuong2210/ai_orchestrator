⚡ Nguyên Tắc Cốt Lõi: Control Plane Is Dumb

Routing là các Python function trong conditional_edges của LangGraph — không có LLM nào quyết định node tiếp theo. Deterministic, testable, auditable. Mọi thất bại đều được route qua Retrospective trước khi đến terminal node FAILED.

🔒 Shell vs Text-Only Mode (Mode B)

Shell agents (engineer, qa, security, deploy) chạy với --dangerously-skip-permissions và cwd=project_dir — có thể đọc/ghi files.
Text-only agents (pm, analyser, reviewer, retrospective, spec_analyze, task_decompose) chạy với --tools "" — chỉ output text, không có tool access.

PM phân loại intent của request, graph route theo path tối ưu — không chạy agents không cần thiết.

🔎 review

SDD Speckit — 5-Pass Spec Validation

🔄
Pass 1 — Duplication
Tìm định nghĩa chồng chéo hoặc mâu thuẫn giữa các artifacts.
❓
Pass 2 — Ambiguity
Tìm statements quá mơ hồ để implement — "handle errors appropriately" không có error codes.
🕳️
Pass 3 — Under-spec
Tìm gaps để lại quyết định cho engineer — missing pagination, no rate limiting policy.
📜
Pass 4 — Constitution
Verify mọi quyết định spec tuân theo immutable project principles.
📊
Pass 5 — Coverage
Verify spec cover tất cả user requirements — mỗi AC maps to ít nhất một task.
🛡️
Server-Side Validation
Agent không thể tự claim approved=True khi có CRITICAL/HIGH finding. Server override.

🔑 job_id = thread_id

run_pipeline tạo job_id (UUID), pass làm thread_id vào LangGraph config. Mọi API call sau đó dùng job_id để lookup đúng checkpoint trong PostgreSQL. job_id = thread_id = checkpoint key.

Rủi Ro & Giới Hạn

Giới Hạn & Rủi Ro

Các rủi ro đã được identify và mitigation strategy tương ứng.

Rủi RoMức ĐộGiải Pháp
QA→Engineer / QA→Analyser loop vô tận Cao MAX_QA_ITERATIONS=3, MAX_QA_ANALYSER_ITERATIONS=2 → FAILED state
Human Gate không có người approve Cao Auto-expire sau HUMAN_GATE_TIMEOUT_HOURS=24, job → FAILED
Prompt injection qua MCP Cao Sanitize input, whitelist actions trước khi forward
Silent failure (hết iteration báo DONE) Cao FAILED terminal state riêng, route qua Retrospective
Agent subprocess timeout (treo mãi) Trung Bình AGENT_TIMEOUT_SECONDS=300 — tự kill subprocess sau 5 phút v5d
Transient subprocess failures Trung Bình Retry 1 lần + 5s backoff trước khi raise error v5d
Cost vượt ngân sách (Mode A) Trung Bình Token budget per agent, warn 80%, hard fail tại 100% v5d
spec_analyze hallucinate approved=True Trung Bình Server-side override: CRITICAL/HIGH findings → force approved=False v5c
Task dependency cycle trong task_decompose Trung Bình DFS cycle detection — parse_submit raises ValueError v5d
Job registry (_jobs) mất khi restart Trung Bình LangGraph state trong Postgres OK — dùng POST /resume/{job_id}
Stale message queue tại terminal nodes Thấp _drain_message_queue() gọi trong handle_done + handle_failed v5d
Mode B throttle (Pro subscription limit) Thấp Monitor usage, switch sang Mode A
